# UI/UX 디자인 원칙 + 시각 표준

## Generic AI 미학 금지 목록

다음은 "AI가 만든 것 같은" 디자인의 특징이다. 피해야 한다:
- 보라/파랑 그라디언트 배경
- 카드 3열 균등 그리드 레이아웃
- 시스템 기본 폰트 (sans-serif, -apple-system 단독)
- 모든 섹션에 아이콘 + 제목 + 설명 3요소 반복
- 과도한 둥근 모서리 (border-radius: 16px 이상)
- 배경 blur 카드 (backdrop-filter 남용)
- "우리 서비스는 ~합니다" 형태의 마케팅 카피 UI

## 타이포그래피 원칙

- Google Fonts에서 디스플레이 폰트(제목용) + 본문 폰트 조합 선택
- 제목은 weight 700–900, 본문은 400–500
- 폰트 크기 계층: 제목(2–4rem) / 소제목(1.25–1.5rem) / 본문(1rem) / 캡션(0.875rem)
- 줄 간격(line-height): 본문 1.6–1.8, 제목 1.1–1.3
- letter-spacing: 제목에 -0.02em 적용 시 고급스러운 느낌

## 색상 원칙

- 지배색(dominant) 1개 + 포인트(accent) 1개 + 중성색(neutral) 조합
- 배경은 순수 흰색(#fff)보다 따뜻하거나 차가운 오프화이트 선호
- 프로젝트 도메인에 맞는 톤: B2B → 절제된 색상 / 소비자 앱 → 생동감 있는 색상
- 명도 대비: WCAG AA 기준 (일반 텍스트 4.5:1, 큰 텍스트 3:1)
- 어두운 배경에 밝은 텍스트보다 밝은 배경이 기본. 다크 테마는 요청 시만

## 공간/레이아웃 원칙

- 대칭보다 비대칭: 중요한 요소를 왼쪽 또는 중앙 정렬, 보조 요소는 작게
- 의도적 여백: 콘텐츠를 숨 쉬게 함. 요소 간 padding은 너그럽게
- 그리드: 균등 분할보다 1:2, 2:3, 황금비 비율 활용
- 콘텐츠 최대 너비: 읽기용 텍스트는 max-width: 65ch, 레이아웃은 max-width: 1280px
- 모바일 퍼스트: 작은 화면에서 시작, 넓어지면 레이아웃 확장

## 복잡도 매칭

미학적 방향에 맞는 코드 복잡도를 선택한다:
- 미니멀/클린 → 단순한 Tailwind 클래스, 애니메이션 절제
- 다이나믹/인터랙티브 → CSS 트랜지션, 호버 효과, 마이크로 인터랙션
- 고급/에디토리얼 → 커스텀 CSS 변수, 정교한 그리드, 스크롤 연동 효과

## 기존 디자인 시스템 우선 규칙

프로젝트에 이미 디자인 시스템이 있는 경우:
1. 기존 색상 토큰, 타이포그래피 스케일, 컴포넌트 라이브러리 우선 사용
2. 새로운 스타일은 기존 시스템과 일관성 유지
3. 기존 규칙을 벗어나야 할 때는 ui-spec.md에 근거 명시
