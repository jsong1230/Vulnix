## 인수조건

### F-01: 저장소 연동 및 스캔 트리거

- [ ] GitHub App으로 설치 후 저장소 목록 조회 및 연동 가능
- [ ] PR 생성 이벤트 수신 시 해당 PR의 변경 파일 대상 자동 스캔 시작
- [ ] 브랜치 푸시 이벤트 수신 시 자동 스캔 시작
- [ ] 웹 UI에서 수동 스캔 트리거 가능
- [ ] 연동 후 첫 실행 시 전체 코드베이스 초기 스캔 수행
- [ ] Webhook 수신 실패 시 재시도 로직 동작 (최대 3회)
- [ ] 연동 해제 시 관련 데이터 정리 처리

### F-02: 취약점 탐지 엔진 (Python)

- [ ] Semgrep 룰 기반 1차 AST 분석으로 후보 취약점 추출
- [ ] Claude API 2차 분석으로 컨텍스트 기반 오탐 필터링
- [ ] SQL Injection 취약점 탐지 가능 (Python 코드 대상)
- [ ] XSS 취약점 탐지 가능 (Python 웹 프레임워크 대상)
- [ ] Hardcoded Credentials 탐지 가능
- [ ] 심각도 분류: Critical / High / Medium / Low / Informational
- [ ] 각 취약점에 OWASP Top 10, CWE ID 매핑
- [ ] 10만 라인 Python 코드베이스 스캔 5분 이내 완료
- [ ] 탐지 정확도 80% 이상 (PoC 기준)

### F-03: 자동 패치 PR 생성

- [ ] 탐지된 취약점별 LLM이 패치 코드 자동 생성
- [ ] 기존 코드 스타일/네이밍 컨벤션 유지
- [ ] 패치 PR에 변경 코드 diff 포함
- [ ] 패치 PR에 취약점 설명 (왜 위험한가) 포함
- [ ] 패치 PR에 패치 설명 (무엇을 어떻게 고쳤는가) 포함
- [ ] 패치 PR에 참고 링크 (CVE, OWASP) 포함
- [ ] 패치 PR에 테스트 코드 제안 포함 (선택적)
- [ ] 패치 불가능한 경우 수동 수정 가이드 + 우선순위 제공
- [ ] 취약점 1건당 패치 PR 생성 30초 이내
- [ ] 패치 PR 승인율 40% 이상 (PoC 기준)

### F-04: 스캔 결과 API 및 기본 UI

- [ ] 스캔 결과 목록 조회 REST API 제공
- [ ] 취약점 상세 정보 조회 API 제공 (심각도, CWE, 코드 위치, 설명)
- [ ] 저장소별 스캔 히스토리 조회 API 제공
- [ ] 웹 UI에서 취약점 목록 확인 가능
- [ ] 웹 UI에서 취약점 상세 (코드 위치, 설명, 패치 PR 링크) 확인 가능
- [ ] 웹 UI에서 수동 스캔 트리거 버튼 동작

### F-05: 다국어 탐지 엔진 확장

- [ ] JavaScript/TypeScript 코드 대상 취약점 탐지 가능
- [ ] Java 코드 대상 취약점 탐지 가능
- [ ] Go 코드 대상 취약점 탐지 가능
- [ ] OWASP Top 10 전체 카테고리 탐지 지원
- [ ] Injection (SQL, Command, LDAP) 전체 유형 탐지
- [ ] XSS (Reflected, Stored, DOM-based) 전체 유형 탐지
- [ ] 인증/인가 취약점 (Insecure JWT, Broken Access Control) 탐지
- [ ] 암호화 취약점 (약한 해시, 하드코딩 키, 안전하지 않은 랜덤) 탐지
- [ ] 설정 오류 (Debug mode, CORS 과도 허용, 민감 정보 로깅) 탐지
- [ ] 언어별 Semgrep 룰셋 커스터마이징 완료

### F-06: 오탐 관리

- [ ] 개발자가 특정 취약점을 "오탐"으로 마킹 가능
- [ ] 오탐 마킹된 패턴을 학습하여 이후 동일 패턴 자동 제외
- [ ] 팀 단위 오탐 규칙 공유 가능
- [ ] 오탐 마킹 해제(복원) 가능
- [ ] 오탐율 트래킹 수치 대시보드에 표시
- [ ] 오탐 피드백이 탐지 엔진 정확도 향상에 반영됨을 확인 가능

### F-07: 대시보드

- [ ] 저장소별 보안 점수 (Security Score) 표시
- [ ] 팀별 보안 점수 집계 표시
- [ ] 취약점 추이 그래프 (신규 발생 / 해결 / 미해결) 표시
- [ ] 심각도별 취약점 분포 차트 표시
- [ ] 취약점 필터링 (저장소, 심각도, 상태, 유형)
- [ ] 취약점 상세에서 패치 PR 링크 바로 이동 가능

### F-08: Slack/Teams 알림 연동

- [ ] Slack Incoming Webhook 설정 가능
- [ ] Microsoft Teams Webhook 설정 가능
- [ ] Critical/High 취약점 탐지 시 설정된 채널로 즉시 알림 발송
- [ ] 알림 메시지에 취약점 요약, 심각도, 패치 PR 링크 포함
- [ ] 주간 보안 리포트 자동 발송 (설정 가능)
- [ ] 알림 수준 커스터마이징 가능 (심각도 임계값 설정)

### F-09: GitLab/Bitbucket 연동

- [ ] GitLab Integration으로 저장소 연동 가능
- [ ] Bitbucket App으로 저장소 연동 가능
- [ ] GitLab Merge Request 이벤트 기반 자동 스캔
- [ ] Bitbucket Pull Request 이벤트 기반 자동 스캔
- [ ] GitLab/Bitbucket 대상 자동 패치 MR/PR 생성
- [ ] 플랫폼 간 통합 대시보드에서 관리

### F-10: CISO 리포트 및 인증 증적

- [ ] CISO용 경영진 리포트 PDF 자동 생성
- [ ] 리포트에 보안 점수 추이, 취약점 통계, 대응 현황 포함
- [ ] CSAP 인증 대응용 증적 자료 포맷 출력
- [ ] ISO 27001 인증 대응용 증적 자료 포맷 출력
- [ ] ISMS 인증 대응용 증적 자료 포맷 출력
- [ ] 리포트 생성 주기 설정 가능 (주간/월간/분기)
- [ ] 리포트 자동 이메일 발송 가능

### F-11: IDE 플러그인 (VS Code)

- [ ] VS Code 익스텐션으로 설치 가능
- [ ] 코드 작성 중 실시간 취약점 하이라이팅
- [ ] 취약점 위치에 인라인 패치 제안 표시
- [ ] 패치 제안 수락 시 코드 자동 수정
- [ ] 취약점 상세 설명 패널 제공
- [ ] Vulnix 서버 연동하여 팀 오탐 규칙 동기화

## 병렬 그룹 규칙

- 같은 마일스톤 내에서만 그룹 구성
- 그룹 내 기능 간 상호 의존성 없음
- 그룹 내 기능 간 충돌 영역 미겹침
- 그룹당 최대 3개 기능

### 병렬 그룹 구성

| 그룹 | 기능 | 설명 |
|------|------|------|
| M1-A | F-01 | 저장소 연동 (선행 작업, 단독 진행) |
| M1-B | F-02 | 탐지 엔진 (F-01 완료 후 진행) |
| M1-C | F-03, F-04 | 패치 PR 생성과 기본 UI (F-02 완료 후 병렬 진행) |
| M2-A | F-05, F-06 | 다국어 확장과 오탐 관리 (병렬 진행) |
| M2-B | F-07, F-08 | 대시보드와 알림 연동 (병렬 진행) |
| M3-A | F-09, F-10 | 멀티 플랫폼과 CISO 리포트 (병렬 진행) |
| M3-B | F-11 | IDE 플러그인 (단독 진행) |
